# 備課教材生成技能（全域）

## 目的
當使用者提到「做備課教材」、「幫我備課」、「生成備課教材」、「做教學素材」、「做授課教材」等需求時，自動啟動本技能，使用 NotebookLM 完成整套備課教材生成流程。

這個技能是**全域技能**，適用於任何專案。

---

## 觸發語句

只要使用者出現下列其中一類說法，就應優先載入本技能：

- 做備課教材
- 幫我備課
- 生成備課教材
- 做教學素材
- 幫我做授課教材
- 根據這份教材做簡報 / 影片 / 音訊 / 資訊圖表
- 根據這份內容做差異化教學教材
- 根據 Bloom 認知層次做教材 / 測驗

---

## 預設工作目標

除非使用者另外縮小範圍，否則預設執行以下完整流程：

1. 建立或選擇 NotebookLM 筆記本
2. 匯入教材來源
3. 生成授課簡報
4. 生成學生自學影片概覽
5. 生成音訊總覽
6. 生成多張資訊圖表
7. 生成 Bloom 差異化教材（記憶、理解、應用、分析）
8. 生成各層次測驗題（每層次 10 題）
9. 將測驗題轉成 CSV 試算表
10. 將差異化教材的 Markdown 報告轉成 Word（DOCX）
11. 下載所有成品到本機資料夾
12. 生成產出物心智圖與素材清單（materials-mindmap.json、materials-inventory.json）

---

## 執行原則

### 1. 資料來源處理

如果使用者提供的是：

- **現成 PDF / 檔案路徑**：優先使用 NotebookLM 的 file source 上傳。
- **貼上的 OCR / 純文字內容**：先整理成結構化 Markdown 來源檔，再上傳。
- **多份教材**：應拆成 2～5 份有結構的來源檔，例如單元總覽、活動教材、補充資料。

整理原則：
- 保留學習目標、重點、活動流程、評量方式、核心概念
- 去除雜訊與重複 OCR 段落
- 標題明確，方便 NotebookLM 理解
- 檔案過大時，優先拆分成數份主題清楚的來源，再匯入 NotebookLM

### 2. 筆記本命名

預設用**學科 + 年級 + 單元主題**建立 NotebookLM 筆記本，例如：

- 健體四下 牙齒保健
- 四年級自然 火山與地震
- 國語 閱讀理解單元

如果已有明確名稱，優先沿用使用者命名。

### 3. 預設輸出位置

所有檔案預設下載到：

```text
C:\Users\shits\Documents\NotebookLM\
```

子資料夾：

```text
slides/
video/
audio/
infographics/
quizzes/
differentiated/
docs/
sheets/
mindmaps/
```

若資料夾不存在，先建立。

### 4. 檔名整理規則

- 若使用者有指定教材代碼、冊別、單元代號或專案前綴，完成生成後應統一加到**所有交付檔名最前面**。
- 若使用者沒有逐字指定，但工作區內已有同系列命名，應優先延續既有序列，不另創新格式。
- 檔名以「**前綴 + 主題 / 層次 / 類型**」為核心結構，確保不同成品可一致搜尋與整理。

建議命名：
- **簡報 / 影片 / 音訊總覽**：`前綴-主題名-檔案類型.副檔名`
- **資訊圖表**：`前綴-圖表主題.png`
- **差異化教材**：`前綴-層次教材.md`、`前綴-層次教材.docx`
- **測驗**：`前綴-層次測驗.json`、`前綴-層次測驗.csv`
- **最終交付清單**：`教學素材/前綴-最終交付清單.md`

若使用者沒有指定前綴，且工作區內也沒有可延續的既有序列，才可保留原始命名。

## 標準流程

### 步驟 A：建立或選擇筆記本

- 列出現有 notebooks
- 若無符合主題者，建立新 notebook
- 回報 notebook 名稱與 notebook ID

### 步驟 B：匯入來源

- 將教材拆整理成 Markdown 或直接上傳檔案
- 使用 NotebookLM source add 匯入
- 等待來源處理完成
- 檢查 source list，確認來源都 ready

### 步驟 C：生成主教材

#### C1. 授課簡報

目標：
- 預設生成詳細版授課簡報
- 若使用者要求至少 20 頁，要在 focus 中明確列出頁面結構與重點清單

#### C2. 學生自學影片

目標：
- 用 explainer 格式
- 適合學生自學
- 內容需涵蓋課程重點

#### C3. 音訊總覽

目標：
- 用 deep_dive
- 適合預習或複習
- 語言使用繁體中文

#### C4. 資訊圖表

目標：
- 每個重點一張
- 預設 landscape
- detail 選 detailed

重點拆分原則：
- 核心概念
- 操作流程
- 常見錯誤或迷思
- 自我檢查／行動建議
- 工具、材料或關鍵辨識項目

### 視覺風格限制說明

- NotebookLM CLI 目前**不能精準控制簡報視覺風格**。
- 若使用者要求粉筆黑板風格，應誠實告知：
  - 簡報只能盡量加強內容結構，無法直接強制成真正黑板風格。
  - 影片可在 style prompt 中描述黑板教學感。
  - 資訊圖表可優先使用最接近的 `sketch_note` 作為替代。

### 步驟 D：生成差異化教材

預設做 4 份：

1. **記憶層次教材**：整理關鍵名詞、事實與基本辨識
2. **理解層次教材**：說明概念、因果與例子
3. **應用層次教材**：設計情境應用與任務實作
4. **分析層次教材**：比較、判斷、分析原因與結果

輸出格式：
- 先生成 report / markdown
- 再另存為 Word（DOCX）

### 步驟 E：生成測驗題

預設每個層次生成 10 題：

- 記憶：10 題
- 理解：10 題
- 應用：10 題
- 分析：10 題

輸出原則：
- 先下載 quiz JSON
- 再轉成 CSV
- CSV 至少包含標準測驗欄位：題目、選項、答案、提示或解析

### 步驟 F：格式轉換

#### F1. Markdown → DOCX

針對差異化教材 `.md`：
- 轉成 `.docx`
- Word 檔完成後，**預設放到 `docs/` 資料夾**，不要留在 `differentiated/`
- 優先保留標題階層、段落、清單、表格

#### F2. Quiz JSON → CSV

將所有 quiz JSON 轉成可開 Excel / 試算表的 UTF-8 CSV。

### 步驟 G：最終整理與交付清單

完成所有生成後，應再執行一次交付整理：

1. 統一檔名前綴（若使用者有指定）
2. 確認差異化教材 DOCX 已移到 `docs/`
3. 保留差異化教材 Markdown 在 `differentiated/`
4. 保留測驗 JSON 與 CSV 在 `quizzes/`
5. 建立一份最終交付清單筆記到 Obsidian

### 步驟 H：生成產出物心智圖與素材清單

在最終整理後，掃描 NotebookLM 輸出目錄，建立結構化的素材清單與心智圖：

**H1. 掃描與分類**
- 掃描 `Documents/NotebookLM/` 目錄下所有產出檔案
- 依類型分類：slides、audio、video、infographics、quizzes、 differentiated、docs、sheets、mindmaps
- 解析檔名提取課程資訊（年級、科目、課名、類型）

**H2. 生成素材清單（materials-inventory.json）**
- 課程資訊（grade, subject, semester, lesson, topic）
- 各類素材列表（檔名、路徑、大小、格式、頁數/時長/題數等）
- 統計摘要（總檔案數、總大小、各類數量）
- 輸出到 `Documents/NotebookLM/materials-inventory.json`

**H3. 生成產出物心智圖（materials-mindmap.json）**
- 以「備課包：課名」為根節點
- 五大分支：📑 教學簡報、🎧 音訊資源、📊 視覺素材、🧠 結構化資料、📝 教師參考
- 各分支下掛載實際檔案節點（含大小、格式、數量）
- 輸出到 `Documents/NotebookLM/materials-mindmap.json`

**H4. 更新 Obsidian 交付清單**
- 在既有交付清單中新增「產出物心智圖」區塊
- 附上素材統計與心智圖檔案路徑
- 提供使用建議（例如：最大檔案、格式建議）

## Obsidian 筆記規則

### 最終交付清單筆記

- 當整套備課教材完成後，應建立一份 Obsidian 筆記作為最終交付清單。
- 這份筆記**預設放在 `教學素材/`**，因為這是教學會直接使用到的輸出，而不是純知識整理。
- 建議檔名格式：`教學素材/前綴-最終交付清單.md`

最終交付清單至少包含：
- 教材主題
- NotebookLM 筆記本名稱與 ID
- 來源數量
- 檔案輸出總路徑
- 各類成品清單
- 本次整理調整
- 使用建議
- 可延伸的後續工作

## 執行時的預設行為

當使用者只說「幫我做備課教材」而沒有列細節時，預設：

- 使用 NotebookLM
- 生成：
  - 授課簡報
  - 學生自學影片
  - 音訊總覽
  - 多張資訊圖表
  - Bloom 4 份差異化教材
  - 每層次 10 題測驗
  - 測驗 CSV
  - 差異化教材 Word 檔
  - 產出物心智圖（materials-mindmap.json）
  - 素材清單（materials-inventory.json）
- 使用繁體中文
- 下載到 Documents/NotebookLM

如果缺少必要資料時，只問**最少問題**：

1. 教材來源是檔案、PDF 還是貼文？
2. 教材主題名稱是什麼？
3. 是否要完整跑全部流程？（若未指定，預設是）

---

## 回報規格

完成後要回報：

1. Notebook 名稱與 ID
2. 已匯入的來源數量
3. 已生成的 artifact 清單
4. 各檔案下載路徑
5. 若有系統限制，要明確說明
6. 若有做檔名整理，也要回報已使用的前綴
7. 若有建立 Obsidian 最終交付清單，要回報筆記路徑
8. 產出物心智圖與素材清單的路徑與統計摘要

---

## 禁止事項

- 不要把 MCP 修復流程混進此技能
- 不要把 OpenCode / NotebookLM 安裝流程混進此技能
- 不要在沒有教材來源時直接硬生成
- 不要隱瞞 NotebookLM 視覺樣式限制
- 不要只生成一部分卻說已完成全流程

---

## 使用方式

使用者只要說「幫我做備課教材」或「根據這份資料做完整備課教材」，就應自動套用本技能。
